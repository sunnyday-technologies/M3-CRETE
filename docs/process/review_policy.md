# Public-Scope Review Policy

## Required owner review

Owner review is required for changes to:

- `README.md`
- `DISCLAIMER.md`
- `SAFETY_NOTICE.md`
- `ELECTRICAL_SCOPE_BOUNDARY.md`
- `index.html`
- `llms.txt`
- `sitemap.xml`
- `bom/**`
- `build-guide/**`
- `blog/**`
- `press/**`
- `config.js` (variant overrides are keyed by BOM item name)
- `docs/commercial/**`
- `docs/process/public_scope_supersession_notice.md`
- Public metadata, SEO descriptions, OpenGraph/Twitter tags, and structured data

## Required professional review

Legal/commercial review is required for terms, disclaimers, commercial
acknowledgements, supersession notices, quote/order language, warranties,
limitations of liability, indemnity, insurance, privacy, and buyer
responsibility language.

Qualified electrical/control-panel review is required for electrical scope,
24 V interface values, controller enclosure boundaries, BOM power items,
professional panel-shop scope, emergency-stop wording, grounding/bonding
wording, current limits, and any text that might be interpreted as electrical
implementation guidance.

Product-safety/mechanical review is required for guarding, pinch/crush hazards,
operator training, lockout/tagout, PPE, cementitious material hazards, and
moving-machine safety language.

## Codex limitation

Codex may implement approved wording, run static checks, and flag
inconsistencies. Codex must not provide final legal, electrical, product-safety,
UL/NRTL/AHJ, OSHA, NEC, NFPA, structural, or field-commissioning sign-off.
